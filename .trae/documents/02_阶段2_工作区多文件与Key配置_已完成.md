# 阶段 2（已完成）：工作区（模拟文件树 + 多文件编辑切换 + API Key 配置）

## 1. 目标
- 从“单文件编辑”升级为“模拟工作区多文件”，补齐 IDE 的基础形态：文件树、打开/切换文件、每文件内容独立保存。
- 将 AI Key 从硬编码改为 UI 配置并持久化，避免泄漏风险并便于演示。

## 2. 范围（In/Out）
**In**
- 模拟工作区（内存 store）：文件树结构 + 多文件内容映射 + active file
- 文件树 UI：递归渲染、折叠/展开、点击打开文件
- 新建文件 / 清空当前文件
- AI Key 设置弹窗：输入/保存到 localStorage
- AI 修改落地：Diff 应用仅作用于当前打开文件

**Out**
- 真实本地文件系统（打开/保存）
- AI 读取多个文件上下文
- 多文件一次性变更审核（multi-file review）

## 3. 主要交互流程
1. 左侧文件树展示模拟工程结构（含 `.gd/.tscn/.md/.html`）。
2. 点击文件 → 中间编辑器切换到该文件内容；编辑会写回该文件。
3. 右侧与 AI 对话 → 若检测到代码 → Diff 审核 → “应用修改”后仅更新当前文件。
4. 预览仅对 `.html/.htm` 文件生效；其他类型显示“不支持预览”占位。
5. Toolbar → 设置 → 输入 API Key → 保存（localStorage 持久化）。
